# Plan: Tablón de Anuncios + Exportación de Notas a PDF

## Feature 1: Tablón de Anuncios del Curso

### Backend

**Schema** — nuevo modelo `announcement` en `schema.prisma`:
```
model announcement {
  id, teachingAssignmentId (FK cascade), title (varchar 150), content (text),
  createdAt, updatedAt, state
}
```
Añadir `announcements announcement[]` al modelo `teachingAssignment`.
→ Migración `prisma migrate dev --name add_announcements`.

**Service** (`academic.service.ts`) — 5 métodos nuevos:
- `findMyAnnouncements(studentId)` — anuncios de los paralelos del alumno
- `findTeacherAnnouncements(teacherId, assignmentId)` — con `assertOwnsAssignment`
- `createAnnouncementByTeacher(teacherId, dto)` — crea con ownership check
- `updateAnnouncementByTeacher` / `deleteAnnouncementByTeacher` — con ownership check
- Añadir `announcements` al include de `findMyClasses` (alumno los ve en su curso)

**Controller** (`academic.controller.ts`) — 5 endpoints:
- `GET /academic/my-announcements` (permiso `academic:student-read`)
- `GET /academic/teacher/announcements/:assignmentId` (permiso `academic:teacher-read`)
- `POST /academic/teacher/announcements` (permiso `academic:teacher-read`)
- `PATCH /academic/teacher/announcements/:id` (permiso `academic:teacher-read`)
- `DELETE /academic/teacher/announcements/:id` (permiso `academic:teacher-read`)

Uso `academic:teacher-read` para los permisos del profesor (evito crear nuevos permisos — el profesor ya tiene ese permiso asignado).

**DTO** — `create-announcement-by-teacher.dto.ts`: `{ teachingAssignmentId, title, content }`.

### Frontend

**Tipos** (`academic.ts`) — `Announcement` type + `MyAnnouncement` type.

**Repositorio** (`apiAcademic.ts`) — 5 métodos: `getMyAnnouncements`, `getTeacherAnnouncements`, `createAnnouncement`, `editAnnouncement`, `deleteAnnouncement`.

**Vista del alumno** (`my-courses/[assignmentId]/page.tsx`):
- Nueva sección "Anuncios" **arriba** de lecciones.
- Cada anuncio como card con título, contenido, fecha (formato relativo "hace 2 días").

**Vista del profesor** (`teacher/assignments/page.tsx`):
- En el `ManageContentDialog`, tercera sección "Anuncios" con crear/editar/eliminar (igual que lecciones).

**Seed** — 2-3 anuncios de ejemplo por profesor.

## Feature 2: Exportación de Notas a PDF

### Frontend (sin backend nuevo — todo client-side)

**Librería:** `jspdf` + `jspdf-autotable` (ya instaladas).

**Ubicación:** Botón "📄 Exportar PDF" en la página `my-grades` del alumno.

**Componente** — `lib/exportGrades.ts`:
- Función `exportGradesToPDF(data: MyGrades, studentName: string)`:
  - Cabecera con nombre del alumno, institución, fecha de generación.
  - Tabla 1: Notas de tareas (Materia, Tarea, Nota, Máx).
  - Tabla 2: Notas de parciales (Materia, Parcial, Nota).
  - Promedio general calculado.
  - Descarga automática con `file-saver` o `doc.save()`.

**Sin endpoints nuevos** — el alumno ya tiene `getMyGrades()` que trae todo.

## Verificación anti-rotura
- No toco auth, ni módulos existentes.
- Los anuncios son un modelo nuevo (no afectan nada existente).
- La exportación es 100% client-side (no toca el backend).
- Sin dependencias nuevas (todo ya instalado).

## Orden de ejecución
1. Backend: schema + migrate + service + controller + DTO
2. Frontend: tipos + repositorio + vista alumno + vista profesor
3. Exportación: lib + botón en my-grades
4. Seed: anuncios de ejemplo
5. Migrate + generate + build + verificar
